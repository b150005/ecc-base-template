# ADR テンプレート

> 英語版: [adr-template.md](./adr-template.md)(原文・Source of Truth)

## このテンプレートの使い方

1. ADR を置く場所を決めます。テンプレートは場所を強制しません。よくある例:
   - 単一言語プロジェクト: リポジトリ直下の `adr/001-use-postgresql.md`
   - 二言語プロジェクト: `adr/en/001-use-postgresql.md` と `adr/ja/001-use-postgresql.md`
   - docs 配下: `docs/adr/001-use-postgresql.md`
2. このファイルを選んだディレクトリにコピーし、`NNN-kebab-case-title.md` にリネームします。
3. 採番は `001` から始めます。テンプレート側で予約された番号はありません。
4. 以下のセクションを埋め、「このテンプレートの使い方」ブロックはコミット前に削除してください。
5. 後から決定が陳腐化した場合は、古い ADR を `Superseded by ADR-NNN` とマークし、新しい ADR を書きます(履歴は書き換えない)。

## コンテキスト

## 決定

何を決めたか。このセクションだけ読んだ人にも選択内容が伝わるように、1〜2 文で簡潔に記述します。

## 帰結

### ポジティブ

- この決定がもたらす利点

### ネガティブ

- 欠点・トレードオフ。この決定によって将来採りにくくなる選択肢は何か

### 中立

- その他の影響(マイグレーションの必要性、チームの作業感、ドキュメント負債など)
